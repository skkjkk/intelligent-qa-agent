### 6.9 登录失败限制功能
**实现方式**：Redis + 原子操作
- Key: `login:fail:用户名`
- Value: 失败次数
- TTL: 10分钟
- 阈值: 5次
- 使用 `increment` 原子操作避免并发问题

**测试结果**：
- 连续5次失败 → 第6次提示"登录失败次数过多，请10分钟后再试" ✅
- 成功登录后 → 清除失败计数 ✅

## 9️⃣ 流式响应Token统计修复（2026-03-24）

### 9.1 发现问题

**问题现象**：流式响应接口Token统计始终为0
```
Token用量 - promptTokens=0, completionTokens=0, totalTokens=0
```

**错误日志**：
```
java.lang.NullPointerException: Cannot invoke "java.lang.Integer.intValue()"
because the return value of "com.jujiu.agent.client.DeepSeekClient$StreamResponse$StreamUsage.getPromptTokens()" is null
```

### 9.2 问题分析

**根本原因**：
1. **Jackson字段映射问题**：DeepSeek API返回JSON使用snake_case（`prompt_tokens`），Java类使用camelCase（`promptTokens`）但缺少`@JsonProperty`注解
2. **NPE防御不足**：API返回的usage字段可能部分为null，代码缺少多层空值检查

**对比发现**：
- 非流式接口正常：`DeepSeekResponse.Usage`类已正确添加`@JsonProperty`注解
- 流式接口异常：`DeepSeekClient.StreamResponse.StreamUsage`类缺少注解

### 9.3 修复内容

#### 修复1：添加Jackson注解（DeepSeekClient.java）
```java
// StreamUsage类添加字段映射
public static class StreamUsage {
    @JsonProperty("prompt_tokens")
    private Integer promptTokens;
    @JsonProperty("completion_tokens")
    private Integer completionTokens;
    @JsonProperty("total_tokens")
    private Integer totalTokens;
}

// StreamChoice类添加字段映射
public static class StreamChoice {
    @JsonProperty("finish_reason")
    private String finishReason;
}
```

#### 修复2：增强NPE防御（ChatServiceImpl.java）
```java
// 安全获取Token用量，防止NPE
Integer promptTokens = result.getUsage() != null && result.getUsage().getPromptTokens() != null
        ? result.getUsage().getPromptTokens() : 0;
```

#### 修复3：安全创建StreamUsage（DeepSeekClient.java）
```java
// 创建安全的StreamUsage对象
StreamResponse.StreamUsage safeUsage = new StreamResponse.StreamUsage(
        usage.getPromptTokens() != null ? usage.getPromptTokens() : 0,
        usage.getCompletionTokens() != null ? usage.getCompletionTokens() : 0,
        usage.getTotalTokens() != null ? usage.getTotalTokens() : 0
);
```

### 9.4 修复原理

**Jackson字段映射规则**：
- 默认按字段名精确匹配（区分大小写）
- `@JsonProperty("snake_case_name")` 可指定JSON字段名
- 流式API响应使用snake_case，Java类使用camelCase，必须显式映射

**DeepSeek流式响应格式**：
```json
// 普通内容消息
{"id":"...","choices":[{"delta":{"content":"Hello"}}]}

// 最后一条消息（包含usage）
{"id":"...","choices":[{"finish_reason":"stop"}],"usage":{"prompt_tokens":10,"completion_tokens":20,"total_tokens":30}}
```
