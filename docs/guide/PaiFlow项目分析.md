## 📖 项目概览

### 基本信息
- **项目名称**：PaiFlow (AI Podcast Workshop - AI播客工坊)
- **项目类型**：基于讯飞AstronAgent二次开发的AI播客生成平台
- **技术栈**：Java + Spring Boot + 微服务架构 + Docker
- **核心功能**：文本转播客稿 → AI语音合成
- **部署方式**：Docker Compose一键部署

### 项目来源
- **基础框架**：讯飞AstronAgent开源项目
- **二次开发**：添加播客生成特定功能
- **开源协议**：Apache 2.0

---

## 🏗️ 系统架构分析

### 微服务架构
```
console-hub (控制台后端)     :8080
console-frontend (前端界面) :1881
core-workflow (工作流引擎)  :7880
core-aitools (AI工具服务)   :18668
nginx (反向代理)           :80
postgres (工作流数据)      :5432
mysql (工具元数据)         :3306
redis (缓存)              :6379
minio (对象存储)          :18998/18999
```

### 数据流向
```
用户输入 → DeepSeek模型 → 播客风格改写 → 讯飞语音合成 → 输出音频
```

### 技术特点
1. **微服务拆分清晰**：控制台、工作流、AI工具分离

3. **容器化部署**：完整的Docker Compose配置
4. **工作流引擎**：可视化工作流编排能力
5. **AI集成**：DeepSeek + 讯飞星火双模型支持

   - 服务间通信方式

   - 工作流状态管理

3. **AI集成模式**
   - 大模型API调用封装
   - 工具调用机制
   - 提示词工程实践

4. **容器化部署**
   - Docker Compose多服务编排
   - 服务依赖管理
   - 环境变量配置

### 不适合初学者的原因
1. **复杂度高**：多服务、多数据库，学习曲线陡峭
2. **代码量大**：需要理解大量现有代码
3. **依赖多**：需要运行多个服务才能正常使用
4. **业务特定**：专注于播客生成，通用性有限

## ⚠️ 注意事项

### 运行要求
- 内存：8GB+
- 磁盘空间：20GB+
- Docker环境：需要安装Docker Desktop

### 可能遇到的问题
1. **服务依赖复杂**：需要同时启动多个服务
2. **API密钥配置**：需要申请DeepSeek和讯飞API
3. **网络问题**：部分服务可能需要外网访问
4. **资源消耗**：运行所有服务占用资源较多

### 学习建议
1. **不要试图一次性理解所有代码**
2. **先运行起来，再逐步深入**
3. **关注核心模块，忽略辅助功能**
4. **做好笔记，记录学习心得**
