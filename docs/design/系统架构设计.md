## 1. 架构概述

### 1.1 架构目标
- **高性能**：响应时间<2秒，支持100+并发
- **可扩展**：模块化设计，易于扩展新功能
- **易维护**：清晰的分层架构，代码规范统一
- **高可用**：故障自动恢复，系统可用性≥99%

### 1.2 架构原则
- 单一职责：每个模块职责明确
- 开闭原则：对扩展开放，对修改关闭
- 依赖倒置：依赖抽象而非具体实现
- 接口隔离：接口设计精简

## 2. 整体架构

### 2.1 分层架构图

```
┌─────────────────────────────────────────┐
│          客户端层（Client Layer）          │
│    Web浏览器 / 移动App / API调用者        │
└─────────────────────────────────────────┘
                    ↓ HTTP/HTTPS
┌─────────────────────────────────────────┐
│        接口层（Controller Layer）         │
│   RESTful API / 参数校验 / 异常处理       │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│        业务层（Service Layer）            │
│  对话管理 / 工具调用 / 工作流编排          │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐

└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│        数据存储层（Data Layer）           │
│      MySQL / Redis / MinIO（可选）       │
└─────────────────────────────────────────┘
```

## 3. 核心模块设计

### 3.1 认证授权模块（Auth Module）

**职责**：
- 用户注册/登录
- JWT Token生成和验证
- 权限控制

**核心类**：
```
controller/AuthController        # 认证接口
service/AuthService              # 认证业务逻辑接口
service/impl/AuthServiceImpl     # 认证业务逻辑实现
security/JwtTokenProvider        # Token生成器
repository/UserRepository        # 用户数据访问
model/dto/request/LoginRequest   # 登录请求DTO
model/dto/response/LoginResponse # 登录响应DTO
```

## 4. 技术架构

### 4.1 技术栈选型

| 层次 | 技术选型 | 说明 |
|------|----------|------|
| 接口层 | Spring MVC | RESTful API |
| 业务层 | Spring Boot | 业务逻辑 |
| 数据访问层 | MyBatis-Plus | ORM框架（MyBatis增强版） |
| 缓存层 | Redis | 分布式缓存 |

| 测试 | JUnit 5 + Mockito | 单元测试 |
| 构建 | Maven | 项目构建 |
| 容器化 | Docker | 容器部署 |

## 6. 数据流转

```
用户输入
  ↓

  ↓
参数校验
  ↓
Service处理业务逻辑
  ↓
检查会话（Redis）
  ↓
调用AI服务（DeepSeek）
  ↓

  ↓
返回响应
```

```
  ↓

  ↓

  ↓
工具执行
  ↓
结果处理
  ↓
返回响应
```

## 7. 安全架构

### 7.1 认证流程

```
用户登录
  ↓
验证用户名密码
  ↓
生成JWT Token
  ↓
返回Token给客户端
  ↓
客户端携带Token访问
  ↓
服务端验证Token
  ↓
允许/拒绝访问
```

### 7.2 安全措施

- **传输安全**：HTTPS加密
- **存储安全**：密码BCrypt加密
- **接口安全**：JWT Token认证
- **防护措施**：限流、防刷、SQL注入防护

## 8. 性能优化

### 8.1 缓存策略

**多级缓存**：
```
本地缓存（Caffeine）
  ↓ 未命中
Redis缓存
  ↓ 未命中
MySQL数据库
```

**缓存内容**：
- 用户信息（TTL: 30分钟）
- 对话历史（TTL: 24小时）
- 工具配置（TTL: 1小时）

### 8.2 异步处理

- 日志记录异步化
- 非关键操作异步化
- 使用线程池管理
