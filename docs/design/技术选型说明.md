## 1. 技术选型原则

### 1.1 选型标准
- **成熟稳定**：选择经过验证的成熟技术
- **社区活跃**：有活跃的社区支持
- **文档完善**：有完整的官方文档
- **易于学习**：学习曲线平缓
- **性能优良**：满足性能要求

## 2. 后端技术栈

### 2.1 Java 17（LTS）

**选型理由**：
- ✅ 长期支持版本（LTS），稳定可靠
- ✅ 性能优化（ZGC垃圾回收器）
- ✅ 新特性支持（Record类、Pattern Matching）
- ✅ 企业广泛采用

**替代方案**：
- Java 21（更新，但社区资源较少）
- Java 11（较旧，缺少新特性）

**最终选择**：Java 17

---

### 2.2 Spring Boot 3.4.4

**选型理由**：
- ✅ 快速开发，开箱即用
- ✅ 自动配置，减少样板代码
- ✅ 内嵌服务器，便于部署
- ✅ 生态完善，插件丰富
- ✅ 官方文档详细
- ✅ 支持响应式编程（WebFlux）

**核心依赖**：
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-webflux</artifactId>
</dependency>
```

**替代方案**：
- Quarkus（更轻量，但生态较小）
- Micronaut（启动快，但学习成本高）

**最终选择**：Spring Boot 3.4.4

---

### 2.3 MyBatis-Plus 3.5.5

**选型理由**：
- ✅ MyBatis 增强版，简化 CRUD 操作
- ✅ 支持自动填充、逻辑删除等高级功能
- ✅ 内置性能分析插件
- ✅ 与 Spring Boot 完美集成

**核心依赖**：
```xml
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-spring-boot3-starter</artifactId>
    <version>3.5.5</version>
</dependency>
```

**替代方案**：
- 原生 MyBatis（更灵活，但样板代码多）
- JPA/Hibernate（更重量级）

**最终选择**：MyBatis-Plus 3.5.5

---

## 3. 数据存储

### 3.1 MySQL 8.0

**选型理由**：
- ✅ 成熟稳定，企业级数据库
- ✅ 事务支持完善
- ✅ 性能优秀
- ✅ 社区活跃

**配置建议**：
```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/intelligent_qa_agent
    username: root
    password: ${DB_PASSWORD}
```

**替代方案**：
- PostgreSQL（功能更强，但学习成本高）
- MariaDB（兼容MySQL）

**最终选择**：MySQL 8.0

---

### 3.2 Redis 7.2

**选型理由**：
- ✅ 高性能缓存
- ✅ 丰富的数据结构
- ✅ 支持持久化
- ✅ 分布式支持

**核心依赖**：
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>

</dependency>
```

**替代方案**：

**最终选择**：Redis 7.2

## 4. 安全框架

### 4.1 Spring Security + JWT

**选型理由**：
- ✅ Spring生态集成
- ✅ 功能完善
- ✅ 安全性高
- ✅ 社区支持好

**JWT库选择**：

## 5. 响应式编程

### 5.1 Spring WebFlux + WebClient

**选型理由**：
- ✅ Spring 5 引入的响应式编程框架
- ✅ 非阻塞 IO，高并发性能优秀
- ✅ 支持流式数据处理（SSE）
- ✅ 与 Spring Boot 完美集成
- ✅ 函数式编程模型，代码简洁

**核心依赖**：
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-webflux</artifactId>
</dependency>
```

**应用场景**：
- 流式响应（AI逐字推送）
- 高并发 API 网关
- 实时数据推送（SSE）

**替代方案**：
- OkHttp（阻塞式，不适合流式）
- Apache HttpClient（较重量级）

**最终选择**：Spring WebFlux + WebClient

## 6. 日志框架

### 6.1 SLF4J + Logback

**选型理由**：
- ✅ Spring Boot默认日志
- ✅ 性能优秀
- ✅ 配置灵活

**配置示例**：
```xml
<configuration>
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>
    <root level="INFO">
        <appender-ref ref="CONSOLE" />
    </root>
</configuration>
```

### 10.2 SSE 流式响应

**选型理由**：
- ✅ Server-Sent Events，服务端推送技术
- ✅ 简单易用，单向通信
- ✅ 支持断线重连
- ✅ 适合 AI 逐字推送场景

**技术实现**：
- 使用 Spring MVC 的 `SseEmitter`
- 配合 WebClient + Flux 实现 DeepSeek 流式调用
- 支持实时"打字机"效果
