## 📌 重要决策记录

1. **前端方案**：不做前端，使用Swagger UI
2. **开发周期**：6周
3. **优先级**：P0（认证、对话）> P1（工具、记忆）> P2（工作流）
4. **API数量**：22个接口
5. **数据库表**：7个MySQL表
6. **协作模式**：AI负责讲解原理、提供思路、审查代码；居九负责自己动手写代码、调试运行、主动思考。AI不直接给完整代码，居九自己编码实践。
7. **工具管理方案**：采用代码注册模式（@Component + ToolRegistry），不维护 Tool 数据表
   - **原因**：工具数量不多，工具信息在代码里，版本控制清晰，减少复杂度
   - **未来扩展**：如需后台动态启用/禁用工具、工具使用统计、工具权限控制，再考虑引入 Tool 表
