## 2. 功能需求

### 2.1 用户注册

**功能编号**：AUTH-001
**功能描述**：新用户注册账号

**输入参数**：
```json
{
  "username": "string (4-20字符，字母数字下划线)",
  "password": "string (8-20字符，需包含大小写字母和数字)",
  "email": "string (有效邮箱格式)",
  "nickname": "string (可选，2-20字符)"
}
```

**处理逻辑**：
1. 参数校验（格式、长度）
2. 检查用户名是否已存在
3. 检查邮箱是否已注册
4. 密码加密存储（BCrypt）
5. 创建用户记录
6. 返回注册结果

**输出**：
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "userId": "string",
    "username": "string",
    "createdAt": "timestamp"
  }
}
```

**验收标准**：
- [ ] 用户名唯一性校验
- [ ] 密码强度校验
- [ ] 邮箱格式校验
- [ ] 密码加密存储
- [ ] 直接返回 Token（自动登录），减少一次请求，是现在主流 APP/网站的做法。

---

### 2.2 用户登录

**功能编号**：AUTH-002
**功能描述**：用户登录获取访问令牌

**输入参数**：
```json
{
  "username": "string",
  "password": "string"
}
```

**处理逻辑**：
1. 验证用户名是否存在
2. 验证密码是否正确
3. 生成JWT Token（有效期24小时）
4. 生成Refresh Token（有效期7天）
5. 记录登录日志
6. 返回Token

**输出**：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "accessToken": "string",
    "refreshToken": "string",
    "expiresIn": 86400,
    "tokenType": "Bearer"
  }
}
```

**验收标准**：
- [ ] 用户名密码验证正确
- [ ] Token生成成功
- [ ] Token包含用户信息
- [ ] 登录失败次数限制（5次/10分钟）

---

### 2.3 Token刷新

**功能编号**：AUTH-003
**功能描述**：使用Refresh Token刷新Access Token

**输入参数**：
```json
{
  "refreshToken": "string"
}
```

**处理逻辑**：
1. 验证Refresh Token有效性
2. 生成新的Access Token
3. 返回新Token

**输出**：
```json
{
  "code": 200,
  "message": "刷新成功",
  "data": {
    "accessToken": "string",
    "expiresIn": 86400
  }
}
```

---

### 2.4 用户登出

**功能编号**：AUTH-004
**功能描述**：用户登出，使Token失效

**输入参数**：
- Header: Authorization: Bearer {token}

**处理逻辑**：
1. 验证Token有效性
2. 将Token加入黑名单（Redis）
3. 返回登出结果

**输出**：
```json
{
  "code": 200,
  "message": "登出成功"
}
```

---

### 2.5 密码修改

**功能编号**：AUTH-005
**功能描述**：用户修改密码

**输入参数**：
```json
{
  "oldPassword": "string",
  "newPassword": "string"
}
```

**处理逻辑**：
1. 验证旧密码
2. 验证新密码强度
3. 更新密码（加密存储）
4. 使所有Token失效
5. 返回修改结果

**验收标准**：
- [ ] 旧密码验证正确
- [ ] 新密码强度符合要求
- [ ] 密码修改后需重新登录

---

### 2.6 密码重置

**功能编号**：AUTH-006
**功能描述**：忘记密码时重置密码

**流程**：
1. 用户请求重置（提供邮箱）
2. 系统发送重置链接到邮箱
3. 用户点击链接设置新密码

**输入参数（步骤1）**：
```json
{
  "email": "string"
}
```

**输入参数（步骤3）**：
```json
{
  "resetToken": "string",
  "newPassword": "string"
}
```

**验收标准**：
- [ ] 重置链接有效期30分钟
- [ ] 重置Token只能使用一次
- [ ] 重置成功后需重新登录

## 4. 数据库设计

### 4.1 用户表（user）

```sql
CREATE TABLE user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(255) NOT NULL COMMENT '密码（加密）',
    email VARCHAR(100) NOT NULL UNIQUE COMMENT '邮箱',
    nickname VARCHAR(50) COMMENT '昵称',
    role VARCHAR(20) DEFAULT 'USER' COMMENT '角色',
    status TINYINT DEFAULT 1 COMMENT '状态：1-正常，0-禁用',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_username (username),
    INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 4.2 登录日志表（login_log）

```sql
CREATE TABLE login_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '用户ID',
    ip VARCHAR(50) COMMENT 'IP地址',
    user_agent VARCHAR(255) COMMENT '用户代理',
    login_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    status TINYINT COMMENT '登录状态：1-成功，0-失败',
    INDEX idx_user_id (user_id),
    INDEX idx_login_time (login_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='登录日志表';
```

## 5. JWT Token设计

### 5.1 Token结构

**Header**：
```json
{
  "alg": "HS256",
  "typ": "JWT"
}
```

**Payload**：
```json
{
  "userId": "string",
  "username": "string",
  "role": "string",
  "iat": 1234567890,
  "exp": 1234654290
}
```

**Signature**：
```
HMACSHA256(
  base64UrlEncode(header) + "." +
  base64UrlEncode(payload),
  secret
)
```

### 5.2 Token配置

- **Access Token有效期**：24小时
- **Refresh Token有效期**：7天
- **签名算法**：HS256
- **密钥存储**：环境变量

## 6. 安全要求

### 6.1 密码安全
- 使用BCrypt加密（强度10）
- 密码最小长度8位
- 必须包含大小写字母和数字
- 不允许使用常见弱密码

### 6.2 Token安全
- Token存储在HTTP-Only Cookie（可选）
- 支持Token黑名单机制
- Token包含过期时间
- 敏感操作需要重新验证

### 6.3 防护措施
- 登录失败限制：5次/10分钟
- 密码重置限制：3次/小时
- IP白名单（可选）
- 异常登录检测
